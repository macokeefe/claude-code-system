Claude Code Session Checklist

Run this before, during, and after every session

Section 1
Pre-Session Setup
Open a fresh Claude Code session (don't continue a context-bloated one)
Run the Project Kickoff prompt — make Claude read your key files first
Tell Claude today's single goal — one thing, not five
Have your error messages / screenshots ready if debugging
Set a time limit — Claude Code works best in focused 45–90 min sprints
Section 2
Mid-Session Warning Signs
⚠ Claude keeps changing things you didn't ask for
Context drift. Run Kickoff prompt again. Make it re-read the current files.
⚠ Answers getting vaguer and more hedged
Context window filling up. Start a fresh session now.
⚠ Claude contradicts code it wrote earlier
Session drift. New session, Kickoff prompt, point it at current file state.
Section 3
End-of-Session Wrap-up
Run Code Review prompt on any changes before committing
Commit your work — don't leave uncommitted changes overnight
Write one sentence in a scratch file: what you accomplished and what's next
Check context — if the session ran long, start fresh tomorrow rather than continuing
