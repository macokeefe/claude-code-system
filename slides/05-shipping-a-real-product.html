The Gap

Works on my machine → People are paying for it

BEFORE SHIP

"Works on my machine"

  • Hardcoded API keys in config
  • No error handling on edge cases
  • Debug logs still printing to console
  • Version is still "1.0.0-dev"
  • Only tested your own happy path
→
AFTER SHIP

"People are paying for it"

  • Credentials in env vars, not source code
  • Every error state has a user-facing message
  • Logs are production-safe
  • Version tagged, deploy config confirmed
  • Tested on real devices and slow connections

Functional Testing

Four tests before every ship

1

Core User Path

Walk the exact sequence a paying customer takes. Sign up → activate → get value. Every click, every screen, in order.

2

Edge Cases

Empty inputs, very long strings, special characters, duplicate accounts, expired sessions. Try to break it before someone else does.

3

Error Recovery

API down, payment fails, file upload times out. Can users recover gracefully, or are they stuck with a blank screen?

4

Performance on Slow Connection

Throttle to 3G in DevTools. Does it still work? Are there loading states? Does it timeout silently?

React Native Checklist

iOS and Android — each has its own gotchas

iOS

  • Info.plist — all permissions declared
  • Bundle ID matches App Store Connect
  • EAS build config verified (eas.json)
  • TestFlight build uploaded and tested
  • App icons all sizes present
  • Privacy manifest if needed (iOS 17+)

Android

  • Package name correct in app.json
  • Signing keystore configured in EAS
  • Play Console internal track uploaded
  • APK/AAB size checked (< 150MB)
  • Adaptive icons configured
  • Target SDK version is current

Launch Sequence

The 7-day launch window

D-7

Teaser Post + Waitlist

Drop a screenshot or demo. "Building X — drop your email for early access." Builds anticipation and gives you day-one buyers.

D0 AM

Gumroad Live + Reddit Posts

Go live at 9am. Email your waitlist. Post in relevant subreddits. Show the thing, don't pitch it.

D0 PM

Twitter/X Thread

The "I built this" thread. Problem → solution → demo → link. Tag relevant people. Real story, not promotion.

D+3

Testimonial / Proof Post

Share an early buyer's result or direct feedback. Social proof closes fence-sitters. Even one real review counts.

D+7

Last-Chance Nudge

"Price goes up at midnight" or "closing early access." Email + social. The simplest revenue spike in the playbook.

First 72 Hours

What to watch and when to act

24h

Traffic vs Conversion Check

Look at Gumroad views vs purchases. Good traffic, no purchases? Conversion problem — rewrite the page. No traffic at all? Distribution problem — get more posts out.

48h

Buyer Feedback Scan

Email every buyer personally. "What nearly stopped you buying?" One email to 5–10 people surfaces every friction point. Fastest market research you'll ever do.

72h

The Decision Point

0 sales: positioning or distribution? 1–5 sales: you have signal — double down on what drove them. 5+ sales: you have a product. Start affiliate outreach now.

Diagnosing Slow Sales

Three different problems, three different fixes

DISTRIBUTION

Not Enough Eyes

Your Gumroad page has very few views. This is not a conversion problem — it's a reach problem.

Fix: More posts, different channels, Reddit communities, DM partnerships.

CONVERSION

Page Isn't Convincing

Traffic is coming in but not buying. High views, low purchases is the signal.

Fix: Rewrite the description, add social proof, clearer outcome statement.

PRODUCT

What Buyers Say

Sales but then refunds. Buyers aren't getting the result the page promised them.

Fix: Read Discord + buyer replies. Ship a v2 built on exactly what they said.
