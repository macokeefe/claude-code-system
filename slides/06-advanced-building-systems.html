Module 06

Building Systems

The difference between using Claude Code and building fast with it.

The Distinction

One-off workflow vs System

ONE-OFF

Starts from scratch each time

  • Re-explains the project every session
  • Relies on memory for conventions
  • Inconsistent code style between sessions
  • First 20 minutes is always context-setting
  • Scope creep goes untracked
SYSTEM

Reusable, documented, faster each project

  • CLAUDE.md auto-loads project context
  • Prompt library means zero cold starts
  • Conventions enforced by the system
  • Every session starts executing immediately
  • Parking lot captures scope creep
CLAUDE.md Template Library

One folder that makes every project faster

Every project type has different defaults. Instead of writing CLAUDE.md from scratch each time, keep a library of battle-tested templates.

  • Copy → paste → fill in project specifics
  • Accumulates your learned conventions over time
  • Universal template handles 80% of projects
  • Stack-specific ones handle the last 20%

The best CLAUDE.md is one you write once and reuse ten times — updated as you learn what Claude needs to know.

# ~/templates/claude-md/ universal.md # base template react-native.md # Expo + RN specifics web-app.md # Next.js / Vite / Vercel api-backend.md # Node + Express / Hono ## universal.md structure: ## Project Overview [one paragraph, what it does] ## Current Focus [what you're building RIGHT NOW] ## Tech Stack [language, framework, key deps] ## Key Conventions [naming, patterns, what to avoid] ## Do Not Touch [files / patterns off limits]
Prompt Library

Five prompts that cover 90% of sessions

KICKOFF

Session Start

Reads CLAUDE.md, confirms current focus, lists the one thing to ship this session.

DEBUG MODE

Error Investigation

Structured error + expected vs actual behavior. No vague "it's broken" prompts.

CODE REVIEW

Pre-Commit Check

Review for bugs, edge cases, style violations, and security issues before committing.

REFACTOR

Clean Up

Specific file or function to clean. "Don't change behaviour, improve clarity and reduce duplication."

FEATURE FROM SPEC

Spec-Driven Build

Paste the user story or spec, ask Claude to confirm understanding before writing a single line. Prevents building the wrong thing.

The Parking Lot

How scope creep gets managed, not ignored

The biggest productivity killer in a Claude Code session is following interesting ideas mid-build. A parking lot captures them without breaking flow.

Rule: When a new idea comes up mid-session, add it to PARKING_LOT.md and keep going. Don't action it. Don't even think about it until the session is done.

  • Protects the session goal
  • No good idea gets lost
  • Becomes your backlog naturally
  • Reviewed at start of next session
# PARKING_LOT.md # Ideas captured mid-session — review next time ## 2026-03-24 - [ ] Add dark mode toggle to settings # came up while building auth flow - [ ] Cache API responses with React Query # noticed repeated fetches in network tab - [ ] Refactor useAuth hook — it's getting long # not urgent, works fine, just messy - [ ] Error boundary component — app crashes on null # URGENT — do this next session
Context Window Management

How to know when to reset — and how to do it

Signs You're Running Low

  • Vague, generic answers instead of specific ones
  • Claude starts repeating earlier suggestions
  • It ignores context you gave earlier
  • Code quality noticeably drops
  • Starts contradicting itself

How to Reset Without Losing Progress

  • Commit what you have first — always
  • Update CLAUDE.md current focus
  • Note key decisions made in the session
  • Start a fresh context window
  • New session reads CLAUDE.md — you're back

The Mid-Session Handoff Prompt

"We're at about 70% context. Before we continue: 1. Summarise what we've built this session 2. List any open decisions 3. What's the next step? I'll start a fresh context with this summary."

Custom Skills

When writing a skill is worth the 20 minutes

🔄

Project-Specific Workflows

You run the same sequence every time you work on a project — generate types, run migrations, sync env. Make it one command.

Signal: You've typed the same prompt three sessions in a row.

📋

Repeated Multi-Step Processes

Anything that takes 5+ back-and-forth exchanges to set up is a candidate. Create a component, add tests, update the index — that's one skill.

Signal: You wrote it in a doc or Notion page already.

👥

Team Standardization

If more than one person needs the same workflow, a skill is better than a Notion doc. Version controlled, always up to date, actually used.

Signal: Someone asked "how do you do X?" twice.
